1 Git

1.1 Git 流程

1.1.1 Git 分为三个区

Git 本地数据管理，大概可以分为三个区，工作区,暂存区和版本库。

  • 工作区（Working Directory）:: 是我们直接编辑的地方，例如 Android Studio 打开的项目，记事本打开的文本等，肉眼可见，直接操作。
  • 暂存区（Stage 或 Index）:: 数据暂时存放的区域，可在工作区和版本库之间进行数据的友好交流。
  • 版本库（commit History）:: 存放已经提交的数据，push 的时候，就是把这个区的数据 push 到远程仓库了。

下面是，当开发者通过 git 修改数据时，各区之间的数据传递流程示意图。

1.2 Git 配置

1.2.1 列出 git 当前配置

git config --list

1.2.2 列出本地配置

git config --local --list

1.2.3 列出全局配置

git config --global --list

1.2.4 列出系统配置

git config --system --list

1.2.5 设置用户名

git config --global user.name “[firstname lastname]”

1.2.6 设置用户邮箱

git config --global user.email “[valid-email]”

1.2.7 设置 git 命令输出为彩色

git config --global color.ui auto

1.2.8 设置 git 使用的文本编辑器设

git config --global core.editor vi

1.3 Git 配置文件

1.3.1 本地仓库配置对应的配置文件路径 [local]

<repo>/.git/config

1.3.2 用户全局配置对应的配置文件路径 [global]

~/.gitconfig

1.3.3 系统配置对应的配置文件路径 [system]

/etc/gitconfig

1.4 创建仓库

1.4.1 复制一个已创建的仓库:

# 通过 SSH
git clone ssh://user@domain.com/repo.git
#通过 HTTP
git clone http://domain.com/user/repo.git

1.4.2 创建一个新的本地仓库:

git init

1.5 本地文件修改

1.5.1 显示工作路径下已修改的文件

git status

1.5.2 显示与上次提交版本文件的不同

git diff

1.5.3 把当前所有修改添加到下次提交中

git add .

1.5.4 把对某个文件的修改添加到下次提交中

git add -p <file>

1.5.5 提交本地的所有修改

git commit -a

1.5.6 提交之前已标记的变化

git commit

1.5.7 附加消息提交

git commit -m 'message here'

1.5.8 提交，并将提交时间设置为之前的某个日期:

git commit --date="date --date='n day ago'" -am "Commit Message"

1.5.9 修改上次提交

请勿修改已发布的提交记录! 

git commit --amend

1.5.10 修改上次提交的 committer date

GIT_COMMITTER_DATE="date" git commit --amend

1.5.11 修改上次提交的 author date

git commit --amend --date="date"

1.5.12 把当前分支中未提交的修改移动到其他分支

git stash
git checkout branch2
git stash pop

1.5.13 将 stashed changes 应用到当前分支

git stash apply

1.5.14 删除最新一次的 stashed changes

git stash drop

1.6 本地文件搜索

1.6.1 从当前目录的所有文件中查找文本内容

git grep "Hello"

1.6.2 在某一版本中搜索文本

git grep "Hello" v2.5

1.7 提交历史

1.7.1 从最新提交开始，显示所有的提交记录（显示 hash， 作者信息，提交的标题和时间）

git log

1.7.2 显示所有提交（仅显示提交的 hash 和 message）

git log --oneline

1.7.3 显示某个用户的所有提交

git log --author="username"

1.7.4 显示某个文件的所有修改

git log -p <file>

1.7.5 仅显示远端<remote/master>分支与远端<origin/master>分支提交记录的差集

git log --oneline <origin/master>..<remote/master> --left-right

1.7.6 谁，在什么时间，修改了文件的什么内容

git blame <file>

1.7.7 显示 reflog

git reflog show

1.7.8 删除 reflog

git reflog delete

1.8 分支与标签

1.8.1 列出所有的分支

git branch

1.8.2 列出所有的远端分支

git branch -r

1.8.3 切换分支

git checkout <branch>

1.8.4 创建并切换到新分支

git checkout -b <branch>

1.8.5 基于当前分支创建新分支

git branch <new-branch>

1.8.6 基于远程分支创建新的可追溯的分支

git branch --track <new-branch> <remote-branch>

1.8.7 删除本地分支

git branch -d <branch>

1.8.8 强制删除一个本地分支

将会丢失未合并的修改！ 

git branch -D <branch>

1.8.9 查看标签

git tag
git tag -l 'v0.1.-'
git show v0.1

1.8.10 给当前版本打标签

git tag <tag-name>

1.8.11 给当前版本打标签并附加消息

git tag -a <tag-name>
git tag -a <tag-name> -m 'my version'

1.9 更新与发布

1.9.1 列出当前配置的远程端

git remote -v

1.9.2 显示远程端的信息

git remote show <remote>

1.9.3 添加新的远程端

git remote add <remote> <url>

1.9.4 下载远程端版本，但不合并到 HEAD 中

git fetch <remote>

1.9.5 下载远程端版本，并自动与 HEAD 版本合并

git remote pull <remote> <url>

1.9.6 将远程端版本合并到本地版本中

git pull origin master

1.9.7 以 rebase 方式将远端分支与本地合并

git pull --rebase <remote> <branch>

1.9.8 将本地版本发布到远程端

git push remote <remote> <branch>

1.9.9 删除远程端分支

git push <remote> :<branch> (since Git v1.5.0)
# 或者
git push <remote> --delete <branch> (since Git v1.7.0)

1.9.10 发布标签:

git push --tags

1.10 合并与重置(Rebase)

1.10.1 将分支合并到当前 HEAD 中

git merge <branch>

1.10.2 将当前 HEAD 版本重置到分支中

请勿重置已发布的提交! 

git rebase <branch>

1.10.3 退出重置

git rebase --abort

1.10.4 解决冲突后继续重置

git rebase --continue

1.10.5 使用配置好的 merge tool 解决冲突

git mergetool

1.10.6 在编辑器中手动解决冲突后，标记文件为已解决冲突

git add <resolved-file>
git rm <resolved-file>

1.10.7 合并提交

git rebase -i <commit-just-before-first>

把下面两段命令的上段内容替换为下段内容

原内容 

pick <commit_id>
pick <commit_id2>
pick <commit_id3>

替换为 

pick <commit_id>
squash <commit_id2>
squash <commit_id3>

1.11 撤销

1.11.1 放弃工作目录下的所有修改

git reset --hard HEAD

1.11.2 移除缓存区的所有文件（i.e. 撤销上次 git add ）

git reset HEAD

1.11.3 放弃某个文件的所有本地修改

git checkout HEAD <file>

1.11.4 重置一个提交（通过创建一个截然不同的新提交）

git revert <commit>

1.11.5 将 HEAD 重置到指定的版本，并抛弃该版本之后的所有修改

git reset --hard <commit>

1.11.6 用远端分支强制覆盖本地分支

git reset --hard <remote/branch> e.g., upstream/master, origin/my-feature

1.11.7 将 HEAD 重置到上一次提交的版本，并将之后的修改标记为未添加到缓存区的修改

git reset <commit>

1.11.8 将 HEAD 重置到上一次提交的版本，并保留未提交的本地修改

git reset --keep <commit>

1.11.9 删除添加 .gitignore 文件前错误提交的文件

git rm -r --cached .
git add .
git commit -m "remove xyz file"

2 Tips

2.1 查看某文件的修改历史 **

先进入此文件所在的目录下

  1. 查看 fileName 相关的 commit 记录： git log filename
  2. 显示每次提交的 diff： git log -p filename
  3. 查看某次提交中的某个文件变化，可以直接加上 fileName： git show commit-id filename
  4. 根据 commit-id 查看某个提交： git show -s --pretty=raw ca53e82

2.2 git log 的常用选项

选项 说明
-p 按补丁格式显示每个更新之间的差异。
–stat 显示每次更新的文件修改统计信息。
–shortstat 只显示 –stat 中最后的行数修改添加移除统计。
–name-only 仅在提交信息后显示已修改的文件清单。
–name-status 显示新增、修改、删除的文件清单。
–abbrev-commit 仅显示 SHA-1 的前几个字符，而非所有的 40 个字符。
–relative-date 使用较短的相对时间显示（比如，“2 weeks ago”）。
–graph 显示 ASCII 图形表示的分支合并历史。
–pretty 使用其他格式显示历史提交信息。
  可用选项 oneline，short，full，fuller 和 format（后跟指定格式）。
-(n) 仅显示最近的 n 条提交
–since,–after 仅显示指定时间之后的提交。
–until,–before 仅显示指定时间之前的提交。
–author 仅显示指定作者相关的提交。
–committer 仅显示指定提交者相关的提交。
–grep 仅显示含指定关键字的提交
-S 仅显示添加或移除了某个关键字的提交
